Implementing xAPI in XR For Improved Data Measurements and Assessment

Stuart Claggett --DIG-IT! Games

Session Description

Achieving measurable results with XR begins with a framework for telemetry data that is interoperable and rich in contextual data. The new Experience Application Programming Interface (xAPI) learning data protocol originally developed by the Department of Defense delivers a standardized way to record, analyze, measure and manage learning experiences. The DOD Instruction 1322.26 (Oct 2017) expects developed or acquired learning products (including XR) to now include xAPI. This session will provide an introduction to what xAPI is and how it can be implemented in XR projects. A sample Unity project will be shown so users can visualize how xAPI data is generated from events in a game/simulation product. An end to end workflow on how to organize your learning and non-learning events and then visualize the data in a business intelligence tool will be demonstrated including AI bots. Research and lessons learned from implementations of xAPI in learning and casual mobile games will be shared. The interoperability benefits of analyzing xAPI data across multiple XR projects is explained. The xAPI protocol is new and early in its adoption by organizations but developers of XR learning products for industry and government need to plan for it now. Large organizations adopting the protocol include Visa, Verizon, Caterpillar, MedStar Health, AT&T and Yum! Brands. Developers aware of the protocol and gaining early experience implementing it, can provide a value add to their XR solutions. Participants will observe how xAPI can be implemented while coexisting with other data solutions and future proof products for later xAPI adoption by clients and stakeholders. Participants will learn about the value and importance of using xAPI in XR projects. They will understand how xAPI can provide rich interoperable contextual data for analyzing the effectiveness of XR products. Session attendees will be provided access to a Unity API and other free tools and resources to accelerate implementation of xAPI in XR projects developed in Unity and other development environments. They will learn about the open source GBLxAPI.org project with free sandbox environments that attendees can use for prototyping.

Bio

Stuart Claggett is the COO/CTO at Dig-iT! Games, a game studio. He leads the open-source xAPI initiative GBLxAPI.org, which aims to simplify and standardize how educational games collect meaningful data for developers, educators, and researchers.
